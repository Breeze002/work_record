# git-push

将当前工作区的改动一次性暂存、提交并推送到远端（origin）。适用于本仓库（work_record）日常的文档更新提交。

## 流程

1. 运行 `git status` 与 `git log --oneline -3`，了解当前改动（新增/修改/删除/重命名）和最近提交风格。
2. 迅速判断本次改动的主题，写一句简洁的中文提交信息。规则：
   - 以动词开头，例如"新增"、"整理"、"更新"、"修复"、"补充"。
   - 概括涉及的内容/模块，不要罗列每个文件名。
   - 不超过 30 字为宜；不要加句号。
   - 参考已有提交风格，如"整理文档：重命名并补充无人机项目迭代记录"、"新增TDL_SDK开发指南/规范及AI求解文档"。
3. 执行：
   ```bash
   git add -A && git commit -m "<提交信息>" && git push
   ```
   合并为一条命令，避免中间停顿。不要加 `Co-Authored-By` 等尾注（这是用户个人记录仓库）。
4. 报告结果：提交哈希、提交信息、推送范围（如 `6b11685..314973f  main -> main`），并用一句话概述本次改动的文件。

## 注意

- 提交者身份是自动推断的 `huifeng.yan@sophgo-inc.com`，无需特殊处理；若用户主动要求修正身份，再提示 `git config --global user.name/user.email`。
- 如果没有任何改动（工作区干净），不要创建空提交，直接告知用户"工作区干净，无需提交"。
- 如果推送失败（如网络/鉴权问题），把报错原样反馈给用户，不要反复重试。
- 中文文件名在 git 输出里会显示成转义八进制，这是正常的，无需处理。
