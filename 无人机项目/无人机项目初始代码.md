# 单目跟踪（SOT）逻辑详解

本文档详细介绍 `tdl_sdk` 中单目标跟踪（Single Object Tracking, SOT）的整体设计、运行逻辑、各组件职责，以及对应代码在仓库中的位置。配合流程图说明，便于快速理解与二次开发。

---

## 1. 概览

SOT 采用 **Siamese（孪生）网络 + 状态机后处理** 的架构：

- **网络层**：FearTrack，一个孪生跟踪网络。输入模板图 + 搜索图，输出目标在搜索图中的响应得分与回归框。
- **跟踪层**：`SOT` 类，负责模板初始化、搜索区裁剪、调用网络推理、卡尔曼滤波平滑、得分统计与遮挡/重现判定、状态机收口。
- **辅助层**：卡尔曼滤波器 `KalmanBoxTracker`、几何/边界工具、分割器（GrabCut/Color/FastSAM）。

核心思想：**tracking-by-matching**。不做全图检测，而是假设目标帧间运动较小，用上一帧的框裁出本帧搜索区，让网络在搜索区里"匹配"目标，得到新框后再作为下一帧的搜索中心，逐帧递推。

## 2. 组件职责

### 2.1 FearTrack 网络（模型层）

- 继承自 `BaseModel`，位于 `src/components/nn/object_tracking/feartrack.cpp`。
- 只负责"给模板 + 搜索图 → 输出目标位置"，**不参与跟踪流程编排**。
- 由 `TDLModelFactory::createObjectTrackingModel(TRACKING_FEARTRACK)` 创建（`tdl_model_factory_internal.cpp:544`）。

**网络输入（2 个分支，孪生结构）**

| 输入 | 尺寸 | 内容 | 生命周期 |
|------|------|------|----------|
| template | `3×128×128` | 首帧裁出的目标外观 | 初始化后固定不变 |
| search | `3×256×256` | 每帧以 `current_bbox_` 为中心裁出的搜索区 | 每帧重裁 |

预处理参数（`feartrack.cpp:59-61`）：`mean={123.675,116.28,103.53}`, `std={58.395,57.12,57.375}`, `rgb`。归一化在模型层完成。

**网络输出（2 个张量）**

| 输出 | 形状 | 含义 |
|------|------|------|
| score map | `16×16` | 每个网格点的响应置信度 |
| regression map | `4×16×16` | 每个网格点距目标左/上/右/下边界的回归量 |

输出尺寸 `16 = instance_size(256) / total_stride(16)`。

**输出解析（`outputParse`, `feartrack.cpp:100-193`）**

1. 在 16×16 score map 上 **argmax** 找响应最大的网格点 `(max_i, max_j)`，得 `max_score`。
2. 取该网格点的 4 个回归通道，结合网格坐标还原 bbox：
   ```
   x1 = grid_x[i][j] - exp(reg_x1)
   y1 = grid_y[i][j] - exp(reg_y1)
   x2 = grid_x[i][j] + exp(reg_x2)
   y2 = grid_y[i][j] + exp(reg_y2)
   ```
   其中 `grid_x[i][j] = j*16`, `grid_y[i][j] = i*16`（`makeGrid`, `feartrack.cpp:80-92`）。
3. 封装为 `ModelBoxInfo{bboxes[0]={x1,y1,x2,y2,score}}`，**坐标系为搜索图 256 坐标系**。

> 框宽高 = `exp(reg_x1)+exp(reg_x2)` / `exp(reg_y1)+exp(reg_y2)`，即网络同时预测"目标在哪"和"目标多大"。

### 2.2 SOT 跟踪器（算法层）

继承自 `Tracker`，位于 `src/components/tracker/sot/sot.cpp`。持有 `sot_model_`（FearTrack，通过 `setModel` 注入）、`kalman_tracker_`、`template_image_`、`current_bbox_` 等状态。负责：

- **初始化** `initialize()` / `initBBox()`：选定目标框、约束边界、建卡尔曼、裁模板图。
- **逐帧跟踪** `track()`：裁搜索图 → 推理 → 映射回原图 → 几何检测 → 得分统计 → 状态判定 → 卡尔曼更新。
- **遮挡/重现判定** `getStatus()`：多信号融合。
- **得分滑窗** `updateScoreLst()`：维护最近 10 帧得分。

> `sot.cpp` **不 include `feartrack.hpp`**，只认 `BaseModel` 抽象接口，与具体网络解耦。

### 2.3 KalmanBoxTracker（平滑/预测层）

位于 `src/components/tracker/sot/kalman_box_tracker.{hpp,cpp}`。对 bbox 做卡尔曼滤波：

- `predict()`：基于运动模型预测下一帧框。
- `update(bbox, true/false)`：`true` 用真实观测修正；`false` 仅预测不修正（遮挡时外推）。
- 由 `use_kalman_filter_` 开关控制是否启用。

### 2.4 分割器（初始化辅助）

当用户只给一个点或检测框不准时，用分割算法精修初始框（`sot.cpp:179-238`）：

| `frame_type` | 算法 | 类 |
|--------------|------|----|
| 1 | GrabCut | `GrabCutSegmentor` |
| 2 | 颜色分割 | `ColorSegmentor` |
| 3 | FastSAM | `FastSAMSegmentor` |

### 2.5 工厂层

- `TrackerFactory::createTracker(TDL_SOT)` → `std::make_shared<SOT>()`（`tracker_factory.cpp:49-57`）。
- `TDLModelFactory::createObjectTrackingModel(TRACKING_FEARTRACK)` → `std::make_shared<FearTrack>()`（`tdl_model_factory_internal.cpp:544`）。
- 装配：`tracker->setModel(sot_model)` 把网络注入跟踪器。

## 3. 整体运行逻辑

### 3.1 装配阶段（一次性）

```
TDLModelFactory::getInstance()
  ├─ loadModelConfig() / setModelDir(model_dir)
  ├─ getModel(sot_model_id)   → FearTrack 网络
  └─ getModel(det_model_id)   → 检测模型（辅助初始化）
TrackerFactory::createTracker(TDL_SOT) → SOT 实例
tracker->setModel(sot_model)            // 网络注入跟踪器
tracker->setUseKalmanFilter(true/false)
```

### 3.2 状态机（示例 `sample/cpp/sample_img_sot.cpp`）

示例用三状态驱动：

```
┌─────────────┐  按 i   ┌──────────────┐ initialize ┌──────────┐
│ DETECTION   │────────►│ WAITING_INPUT │──────────►│ TRACKING │
│ (跑检测打印) │         │ (读用户框/点)  │            │ (每帧track)│
└─────────────┘         └──────────────┘            └────┬─────┘
                          ▲                              │
                          │ 丢失≥5秒                      │ status=LOST
                          └──────────────────────────────┘
```

- **DETECTION**（`processDetectionState`, `sample:80-100`）：只跑检测器打印框。
- **WAITING_INPUT**（`processWaitingInputState`, `sample:103-169`）：读用户输入（点 `x,y` 或框 `x1,y1,x2,y2`），跑一次检测，调 `tracker->initialize()`。
- **TRACKING**（`processTrackingState`, `sample:172-207`）：每帧调 `tracker->track()`，输出框；连续丢失 5 秒回到 DETECTION。

### 3.3 初始化流程 `initialize()` → `initBBox()`（`sot.cpp:173-555`）

```
输入：image + 检测框列表 + 目标框(或点) + frame_type
  │
  ├─ 检测框为空？─是─► 按 frame_type 调分割器(GrabCut/Color/FastSAM)精修框
  │                   └─ 失败回退到原始 bbox
  │
  └─ 检测框非空 ──► 取中心落在目标框内、距离最近的检测框
  │
  ▼
initBBox(image, bbox):
  ├─ 重置状态：score_lst_/lost_frames_/status_=TRACKED ...
  ├─ clampBBox：约束框到图像边界内（最小边3）
  ├─ 设 current_bbox_、last_reliable_template_bbox_
  ├─ if use_kalman_filter_: kalman_tracker_ = KalmanBoxTracker(current_bbox_)
  └─ preprocess 裁 128×128 模板图 template_image_（offset=0.2，整段跟踪固定）
```

### 3.4 逐帧跟踪流程 `track()`（`sot.cpp:557-777`）

```
① preprocess 裁 256×256 搜索图
     以 current_bbox_ 为中心，offset=search_bbox_offset_(3.0) 外扩
     context = [x-3w, y-3h, 7w, 7h]，resize 到 256
                          (sot.cpp:565-566)

② 组输入 input_images = {{template_image_, search_image}}   (sot.cpp:568)

③ sot_model_->inference(input_images, output_datas)         (sot.cpp:572)

④ outputParse → track_result.bboxes[0]={x1,y1,x2,y2,score}
     坐标系为搜索图 256 坐标系                            (feartrack.cpp:100-193)

⑤ Kalman 预测
     kalman_bbox = kalman_tracker_->predict()
     若跳帧(frame_id - frame_id_ > 1)：连续 predict 补齐   (sot.cpp:601-609)

⑥ 映射回原图
     w_scale = context[2]/256, h_scale = context[3]/256
     scaled_bbox = {x1*w_scale+context[0], y1*h_scale+context[1],
                    w*w_scale, h*h_scale}
     clampBBox 约束边界                                  (sot.cpp:611-619)

⑦ 几何异常检测
     中心位移 > 2×宽  或  面积比/宽高比 突变(>1.25 或 <0.8)
     → 异常：status=LOST，直接返回                       (sot.cpp:621-658)

⑧ 不用 Kalman 分支：score<0.5 → LOST，直接返回           (sot.cpp:663-677)

⑨ 计算 iou（网络框 vs 卡尔曼预测框）
     Kalman 预热 25 帧后才启用，否则 iou=1.0, kalman_bbox=bbox  (sot.cpp:680-694)

⑩ Score 统计：updateScoreLst(score) → score_ratio_        (sot.cpp:696)

⑪ 状态判定：getStatus(bbox, kalman_bbox, score, score_ratio_, iou, size_ratio)
     → is_occluded / is_reappear                          (sot.cpp:705)

⑫ 状态机收口 + Kalman 更新                              (sot.cpp:715-769)
     assert_occluded = (score<0.1 || iou<0.2) → LOST
     TRACKED & 未遮挡  → kalman.update(bbox, true),  lost_frames=0
     TRACKED & 遮挡    → status=LOST, kalman.update(kalman_bbox,false), lost_frames=1
     LOST   & 重现     → status=TRACKED, kalman.update(bbox,true)
     LOST   & 未重现   → kalman.update(kalman,false), lost_frames++,
                         搜索区按 lost_frames 外扩(最多2倍)

⑬ 输出 tracker_info.box_info_ + status_
     兜底：is_occluded && lost_frames>3 → LOST            (sot.cpp:771-773)
```

### 3.5 端到端数据流总览

```
用户参数(model_dir, sot_id, det_id, image_folder)
        │
        ▼
工厂创建 FearTrack + SOT + 检测器
        │
        ▼
┌──────────────────────── 主循环（逐帧）────────────────────────┐
│  readImage(frame)                                              │
│      │                                                         │
│      ▼                                                         │
│  按状态派发                                                    │
│      │                                                         │
│      ├── DETECTION ──► det.inference → 打印框                  │
│      │                                                         │
│      ├── WAITING_INPUT ─► 读用户框/点 → det → initialize()     │
│      │                    (裁模板、建Kalman) → TRACKING         │
│      │                                                         │
│      └── TRACKING ─► track():                                  │
│            裁搜索图(template固定) → inference → outputParse(256框)│

│            → Score+getStatus判遮挡/重现 → Kalman update          │
│            → 更新 current_bbox_                                 │
│            → tracker_info{box, score, status}                   │
│                                                                 │
│  status==LOST? → 丢失计时 → ≥5秒回 DETECTION                   │
└─────────────────────────────────────────────────────────────────┘
```

## 4. 跟踪目标的递推机理

`track()` 是**有状态**的，每次调用复用上一次的结果：

| 状态变量 | 作用 | 更新时机 |
|----------|------|----------|
| `template_image_` | 目标外观锚点 | 初始化时赋值，全程不变 |
| `current_bbox_` | 本帧输出 + 下帧搜索中心 | 每帧 `track()` 末尾更新（`sot.cpp:661`） |
| `kalman_tracker_` | 运动预测/平滑 | 逐帧 predict/update |
| `score_lst_` | 最近 10 帧得分滑窗 | 每帧 `updateScoreLst` 滚动 |
| `last_reliable_template_bbox_` | 最近可靠框 | 正常跟踪时更新，丢失时作为外扩搜索中心 |

**递推本质**：每帧拿上一帧的 `current_bbox_` 裁新的搜索图、用同一个模板图去匹配、得到新框、再更新 `current_bbox_`，循环往复。

## 5. "目标是否在画面中"的判定

> 重要前提：SOT **不做全图检测**，只判定"目标在不在以 `current_bbox_` 为中心的搜索区内"。目标跑出搜索区（快速运动或长时间遮挡漂走）也会被判丢失。

判定分多道关卡，按执行顺序：

### 关卡 1：网络有无输出（`sot.cpp:586-590`）
score map argmax 找不到有效网格点 → `bboxes.empty()` → `LOST`。最硬的"找不到"信号。

### 关卡 2：几何异常（`sot.cpp:621-658`）
```
中心位移 > 2 × 当前宽        → 异常
面积比 > 1.25 或 < 0.8       → 异常（尺度突变）
宽高比 > 1.25 或 < 0.8       → 异常（形变突变）
```
任一成立 → `LOST`。防止网络在遮挡时乱跳到背景。

### 关卡 3：得分/IoU 快判
- 不开 Kalman：`score < 0.5` → `LOST`（`sot.cpp:672`）。
- 开 Kalman：`score < 0.1 || iou < 0.2` → `assert_occluded` → `LOST`（`sot.cpp:715`）。`iou` 为网络框与卡尔曼预测框的 IoU。

### 关卡 4：getStatus 多信号融合（核心，`sot.cpp:18-57`）

**输入信号：**

| 信号 | 含义 | 来源 |
|------|------|------|
| `score` | 网络响应强度 | FearTrack argmax |
| `score_ratio` | 当前 score / 最近 10 帧均值 | `updateScoreLst`（`sot.cpp:149-161`） |
| `iou` | 网络框 vs 卡尔曼预测框 | `MotBoxHelper::calculateIOU`，预热 25 帧后启用 |
| `size_ratio` | 当前 w/h 比 / 上一帧 w/h 比 | `sot.cpp:698-703` |

**遮挡置信度**（得分低 + 运动不一致 → 疑似遮挡）：
```
confidence_of_occluded = 0.1*score_abs        // score < 0.6 的差距
                       + 0.7*score_ratio_abs  // score_ratio < 0.9 的差距（权重最大）
                       + 0.5*iou_abs          // iou < 0.8 的差距
                       + 0.5*size_ratio_abs   // 尺度比偏离 1
> occluded_threshold_(0.3) → is_occluded = true
```

**重现置信度**（得分回升 + 运动恢复 → 疑似重现）：
```
confidence_of_reappear = 2*reappear_score_abs        // score > 0.3
                       + 2*reappear_score_ratio_abs  // score_ratio > 0.3
                       + 0.1*reappear_iou_abs         // iou > 0.3
                       - 0.2*size_ratio_abs
> reappear_threshold_(2) → is_reappear = true
```

阈值参数定义在 `sot.hpp:90-99`。

### 关卡 5：状态机收口（`sot.cpp:721-777`）

```
当前 TRACKED：
  is_occluded = true  → status=LOST, Kalman 仅预测(不修正), lost_frames=1
                       搜索区按 lost_frames 外扩(最多2倍)  (sot.cpp:728-732, 751-754)
  is_occluded = false → Kalman 用真实观测更新, lost_frames=0  (sot.cpp:734)

当前 LOST：
  is_reappear = true  → status=TRACKED, Kalman 恢复更新          (sot.cpp:741-747)
  is_reappear = false → Kalman 继续仅预测, lost_frames++,
                       搜索区继续外扩                            (sot.cpp:748-768)

兜底：is_occluded && lost_frames > 3 → LOST                     (sot.cpp:771)
```

### 判定流程图

```
网络推理
   │
   ▼
bboxes 为空？ ─是─► LOST
   │否
   ▼
几何异常？ ─是─► LOST
   │否
   │
   ├─ 未开 Kalman ─► score<0.5? ─是─► LOST
   │                  否
   ├─ 开 Kalman ───► score<0.1 || iou<0.2? ─是─► LOST(assert_occluded)
   │                  否
   ▼
getStatus 融合 score/score_ratio/iou/size_ratio
   │
   ├─ is_occluded  ──► LOST + Kalman仅预测 + 搜索区外扩
   ├─ is_reappear  ──► TRACKED + Kalman恢复更新
   │
   ▼
is_occluded && lost_frames>3? ─是─► LOST
   │否
   ▼
输出 TRACKED + 框
```

## 6. 丢失后的恢复机制

判定丢失后**不立即放弃**，进入"预测 + 外扩搜索"待机态（`sot.cpp:748-768`）：

```
expand_ratio = min(1 + lost_frames/20, 2.0)   // 丢得越久搜得越宽，最多2倍
```

- Kalman 只预测不修正，维持轨迹，假设目标仍在附近移动。
- 搜索区以 `last_reliable_template_bbox_` 为中心逐帧外扩。
- 期间每帧仍跑网络，一旦 `is_reappear`（score 回升 + IoU 恢复）→ 切回 TRACKED 并恢复卡尔曼更新。

只有连续丢失超过阈值（示例中再加 5 秒超时，`sample:200-204`）才回到 DETECTION 态，等用户重新选目标。

## 8. 一句话总结

> 用户给"目标框 + 图像序列"，SOT 第一帧用框裁模板并初始化（建卡尔曼），之后每帧用上一帧的框裁搜索图、FearTrack 网络匹配出 256 坐标系下的新框、映射回原图，再用 Kalman 平滑、Score 滑窗 + 几何/IoU 多信号融合判定遮挡/重现，把新框同时作为下一帧的搜索中心——如此递推，直到丢失超时回到检测态。网络管"在哪"，Kalman+Score 管"信不信、丢没丢"。

## 一、模块定位与整体关系

SOT 是 `Tracker` 抽象基类的两个实现之一（另一个是多目标 `MOT`）。它解决**单目标长时跟踪**问题：用户在首帧给定一个目标（框/点/检测索引），后续每帧输出该目标的边界框与状态（TRACKED / LOST）。

```
                    TrackerFactory::createTracker(TrackerType)
                                │
                ┌───────────────┴───────────────┐
                ▼ TDL_MOT_SORT                  ▼ TDL_SOT
              MOT (多目标)                    SOT (单目标)
                                                │
              ┌────────────── SOT 内部依赖 ──────────────┐
              │                                          │
   sot_model_ (FearTrack, BaseModel)          kalman_tracker_ (KalmanBoxTracker)
   双输入孪生网络推理                         8维状态卡尔曼滤波(可选)
              │                                          │
              ▼                                          ▼
   preprocess(模板/搜索区域)                  predict/update
              │
              ▼
   目标框选 (initialize 阶段, 可选):
   ├─ GrabCutSegmentor   (frame_type=1, OpenCV, 耗时高)
   ├─ ColorSegmentor     (frame_type=2, 颜色阈值)
   └─ FastSAMSegmentor   (frame_type=3, 分割模型)
   └─ frame_type=0       直接用原始框
```

模型类型：`ModelType::TRACKING_FEARTRACK`（C 侧 `TDL_MODEL_TRACKING_FEARTRACK`），由 `TDLModelFactory` 创建，C API 内部固定使用该模型。

### 2.4 状态成员

## 三、初始化流程

SOT 提供 3 个 `initialize` 重载，对应 3 种目标指定方式。C API `TDL_SetSingleObjectTracking` 根据 `set_values` 的 `size` 分发：
- `size==1` → index 模式（检测框索引）
- `size==2` → point 模式 `(x,y)`
- `size==4` → bbox 模式 `(x1,y1,x2,y2)`

### 3.1 三种重载的共同目标

最终都调用 `initBBox(image, final_bbox)` 完成真正的初始化。区别在于"如何从用户输入 + 检测框得到 `final_bbox`"。

### 3.2 bbox 模式 `initialize(image, detect_boxes, bbox, ...)`

```
1. 若 detect_boxes 为空 → 直接走 frame_type 框选(见 3.5)，无框选则 initBBox(image, bbox)
2. 否则在 detect_boxes 中找"中心点落入 bbox 内"且距 bbox 中心最近的检测框
   └─ 用该检测框作为 final_bbox (比用户手画框更紧致)
3. 若无重叠检测框 → 走 frame_type 框选
```

### 3.3 point 模式 `initialize(image, detect_boxes, x, y, ...)`

```
1. 若 detect_boxes 为空 → 必须用 frame_type 框选(frame_type=0 时报错返回-1)
2. 否则找"包含点(x,y)"的检测框 → initBBox(image, detect_box)
3. 若无包含该点的检测框 → 走 frame_type 框选
```

### 3.4 index 模式 `initialize(image, detect_boxes, index, ...)`

```
最简单: 检测框非空且 index 合法 → initBBox(image, detect_boxes[index])
不涉及 frame_type 框选。
```

### 3.5 目标框选算法（frame_type）

当无可用检测框或用户希望更紧致的目标区域时，用种子点（bbox 中心或用户点）调用框选器，得到比粗框更贴合目标的 `area_bbox`：

| frame_type | 算法 | 实现 | 特点 |
|------------|------|------|------|
| 0 (TDL_REJECT) | 不框选 | — | 直接用原框 |
| 1 (TDL_GRABCUT) | GrabCut | `GrabCutSegmentor` | OpenCV 迭代分割，多半径{3,5,11,13}试探，IoU<0.6 才接受（即分割框要明显小于种子圆），耗时高 |
| 2 (TDL_COLOR) | 颜色阈值 | `ColorSegmentor` | 以种子点邻域均值为颜色原型，inRange + 连通域，多 diff 试探，面积/中心约束 |
| 3 (TDL_FASTSAM) | FastSAM 分割 | `FastSAMSegmentor` | 以 seed 为中心抠 320×320 跑 `FASTSAM_SEG`，取包含 seed 的最小目标框 |

框选失败时**降级**为 `initBBox(image, bbox)`（bbox 模式）或返回 -1（point 模式）。`FastSAMSegmentor` 在 SOT 内以成员变量缓存，避免重复构造。

### 3.6 `initBBox` 真正的初始化

```
initBBox(image, init_bbox):
  1. 重置状态: status_=TRACKED, lost_frames_=0, score_lst_.clear(),
     score_ratio_=1.0, prev_w_h_ratio_=0, sot_info_ 清零
  2. 转 [x,y,w,h], clampBBox 边界约束(min_side=3)
  3. current_bbox_ = last_reliable_template_bbox_ = sot_info_.template_bbox = 框
  4. (可选) use_kalman_filter_ → new KalmanBoxTracker(current_bbox_)
  5. preprocess(image, current_bbox_, offset=template_bbox_offset_,
                crop_size=template_size_, context) → template_image_
     └─ 模板图: 以当前框为中心, 按 offset 外扩后裁剪到 template_size_×template_size_
  6. is_initialized_ = true
```

注意：**模板图只在初始化时提取一次，后续跟踪不再更新**。这是相对保守的策略（FEARTrack 类的典型用法），优点是稳定，缺点是目标外观剧烈变化时模板过时。

## 四、跟踪流程（核心）

`SOT::track(image, frame_id, tracker_info)` 每帧调用。完整步骤：

```
track(image, frame_id, tracker_info):
┌─────────────────────────────────────────────────────────────────┐
│ Step 1. 提取搜索图                                              │
│   search_image = preprocess(image, current_bbox_,               │
│              offset=search_bbox_offset_, crop_size=instance_size_,│
│              context)                                           │
│   └─ 以当前框为中心, 外扩 3.0 倍后裁剪到 256×256                │
│      context 记录实际裁剪区域 [cx,cy,cw,ch] (用于坐标还原)     │
├─────────────────────────────────────────────────────────────────┤
│ Step 2. 双输入推理                                              │
│   sot_model_->inference({{template_image_, search_image}}, out) │
│   └─ FearTrack: 模板+搜索 → score_map(16×16) + reg_map(4×16×16)│
│   取 out[0] 为 ModelBoxInfo (track_result)                      │
│   若 bboxes 空 → tracker_info.status_=LOST, return             │
├─────────────────────────────────────────────────────────────────┤
│ Step 3. 坐标映射回原图                                          │
│   模型输出框基于 instance_size_ 空间:                          │
│   w_scale = context[2] / instance_size_                        │
│   h_scale = context[3] / instance_size_                        │
│   scaled_bbox = [x1*w_scale+context[0], y1*h_scale+context[1], │
│                  w*w_scale,        h*h_scale]                   │
│   clampBBox(scaled_bbox)                                         │
├─────────────────────────────────────────────────────────────────┤
│ Step 4. 几何异常检测 (硬阈值, 无卡尔曼也生效)                  │
│   center_displacement = ‖new_center - prev_center‖             │
│   area_ratio   = new_area / prev_area                          │
│   aspect_ratio = new_aspect / prev_aspect                      │
│   geom_anomaly = 位移 > 2*prev_width                           │
│               || area_ratio ∉ [0.8, 1.25]                      │
│               || aspect_ratio ∉ [0.8, 1.25]                    │
│   若异常 → status_=LOST, 输出旧框, return (不更新 current_bbox_)│
├─────────────────────────────────────────────────────────────────┤
│ Step 5. 更新 current_bbox_ = scaled_bbox                        │
├─────────────────────────────────────────────────────────────────┤
│ Step 6. 【无卡尔曼分支】                                        │
│   输出 scaled_bbox + score, status_=TRACKED                    │
│   score < 0.5 → status_=LOST                                    │
│   return                                                        │
├─────────────────────────────────────────────────────────────────┤
│ Step 7. 【有卡尔曼分支】                                        │
│   7a. kalman_tracker_->predict() → kalman_bbox                 │
│       若帧间隔>1, 补做 (frame_id-frame_id_-1) 次 predict+update │
│       (跳帧补偿)                                                │
│   7b. 计算 IoU:                                                 │
│       if update_count_ > kalman_update_count_(25):              │
│         iou = IoU(观测框, kalman框)                            │
│       else:                                                     │
│         iou = 1.0, kalman_bbox = 观测框  (前25帧信任观测)       │
│   7c. updateScoreLst(score): 维护近10帧队列,                    │
│       score_ratio = score / avg(score_lst_)                    │
│       (仅当 score_ratio <= 阈值才入队, 防遮挡拉低基线)         │
│   7d. size_ratio = (w/h)_cur / prev_w_h_ratio_                 │
│   7e. getStatus(): 计算 occluded/reappear 置信度 (见第五章)    │
│   7f. 输出 scaled_bbox + score, status_=TRACKED                │
│   7g. assert_occluded = (score<0.1 || iou<0.2)                 │
│       若 assert_occluded → status_=LOST                        │
│   7h. 状态机更新 (见第五章)                                    │
└─────────────────────────────────────────────────────────────────┘
最终: tracker_info{box_info_(scaled), status_}, 更新 frame_id_
```

### 4.1 预处理细节 `preprocess(image, bbox, offset, crop_size, context)`

```
context[0] = x - w*offset      // 裁剪左上 x
context[1] = y - h*offset      // 裁剪左上 y
context[2] = w*(1+2*offset)    // 裁剪宽
context[3] = h*(1+2*offset)    // 裁剪高
→ clamp 到图像内
PreprocessParams: crop=上述区域, dst=crop_size×crop_size,
                  keep_aspect_ratio=false, mean=0, scale=1
preprocessor_->preprocess(image, params, nullptr) → crop_image
```

预处理平台 `AUTOMATIC`（由 `PreprocessorFactory` 按编译平台选 VPSS/OpenCV/BMCV）。模板/搜索图均保持原图格式与数据类型。

### 4.2 `updateScoreLst` 的防遮挡基线机制

```
if 队列满(>=10):
  avg = mean(score_lst_)
  score_ratio = score / avg  (avg>0 时)
  if score_ratio > occluded_score_ratio_threshold_(0.9):
    # 当前得分相对历史不算太低 → 才入队更新基线
    pop_front, push_back(score)
else:
  push_back(score)
```

含义：当目标开始遮挡、得分骤降时，`score_ratio` 会 ≤ 0.9，**不更新历史队列**，从而保留遮挡前的正常得分基线，使 `score_ratio` 持续偏小，作为遮挡判据。这是该实现的一个精巧点。

## 五、遮挡 / 重现状态机

`getStatus(bbox, kalman_bbox, score, score_ratio, iou, size_ratio)` 计算两个加权置信度：

```
# 各项"差值"(均 clamp 到 >=0, 越大越异常)
score_abs            = max(occluded_score_threshold_     - score,        0)   # 0.6
score_ratio_abs      = max(occluded_score_ratio_threshold_ - score_ratio,0)  # 0.9
iou_abs              = max(occluded_iou_threshold_       - iou,          0)  # 0.8
size_ratio_abs       = |size_ratio - size_ratio_threshold_|                  # |..-1|

confidence_of_occluded  = 0.1*score_abs + 0.7*score_ratio_abs
                        + 0.5*iou_abs   + 0.5*size_ratio_abs
is_occluded = confidence_of_occluded > occluded_threshold_(0.3)

# 重现: 各项"超出"(越大越像重现)
reappear_score_abs       = max(score - reappear_score_threshold_,       0)   # 0.3
reappear_score_ratio_abs = max(score_ratio - reappear_score_ratio_threshold_,0)#0.3
reappear_iou_abs         = iou - reappear_iou_threshold_                      # 0.3
confidence_of_reappear = 2*reappear_score_abs + 2*reappear_score_ratio_abs
                       + 0.1*reappear_iou_abs - 0.2*size_ratio_abs
is_reappear = confidence_of_reappear > reappear_threshold_(2)
```

随后在 `track` 末尾执行状态转移：

```
state = TRACKED (本帧判定)
if assert_occluded(score<0.1 || iou<0.2):  status_ = LOST

if status_ == TRACKED:
    if is_occluded:
        status_ = LOST
        kalman.update(kalman_bbox, update_scale=false)   # 仅预测不更新尺度
        last_reliable_template_bbox_ = current_bbox_ = template_bbox = scaled_bbox
        lost_frames_ = 1
    else:
        kalman.update(bbox, update_scale=true)            # 观测更新
        last_reliable_template_bbox_ = current_bbox_ = scaled_bbox
        lost_frames_ = 0
else:  # status_ == LOST
    if (not assert_occluded) and is_reappear:
        status_ = TRACKED
        kalman.update(bbox, update_scale=true)
        current_bbox_ = scaled_bbox
        lost_frames_ = 0
    else:
        kalman.update(kalman_bbox, update_scale=false)
        lost_frames_ += 1
        # 搜索框逐帧扩张 (最多到 max_expand_ratio_=2.0)
        expand_ratio = min(1 + lost_frames_/20, max_expand_ratio_)
        以 last_reliable_template_bbox_ 中心为锚, 等比扩张 → template_bbox/current_bbox_

# 最后: 若 is_occluded 且 lost_frames_ > 3 → 强制 status_=LOST
```

状态机要点：
- **遮挡判定主导信号是 `score_ratio_abs`（权重 0.7）**，即"得分相对历史的下降"比绝对得分更关键。
- 丢失期间**搜索框线性扩张**（每 20 帧扩张 1 倍，上限 2 倍），增加重捕获概率。
- **重现判定阈值很高**（`reappear_threshold_=2`，且要求非 `assert_occluded`），避免误重捕获到背景。
- 模板图始终不变；丢失时 `current_bbox_` 被设为扩张后的 `template_bbox_`，下一帧搜索区域据此扩张。

## 六、卡尔曼滤波 `KalmanBoxTracker`

8 维常速度模型：状态 `[cx, cy, w, h, vx, vy, vw, vh]`。

```
F (转移) = I_8, 且 F[0,4]=F[1,5]=F[2,6]=F[3,7]=1   (位置 += 速度)
H (测量) = [I_4 | 0]   (只观测 cx,cy,w,h)
Q (过程噪声) = I_8 * 0.001
R (测量噪声) = I_4 * 0.2
初始化: 状态=(cx,cy,w,h,0,0,0,0), 误差协方差=I_8, update_count_=1
```

- `predict()`：标准卡尔曼预测。**高度强制按 `last_reliable_width_/last_reliable_height_` 的宽高比由预测宽度反推**，避免尺度漂移导致宽高比失真。
- `update(bbox, update_scale)`：
  - `update_scale=true`（用观测/检测框）：完整 4 维更新，刷新 `last_reliable_width/height`。
  - `update_scale=false`（用预测框/丢失期）：**临时把 H 改成只测位置**（cx,cy），更新后强制恢复旧 w,h 并清零 vw,vh，保持尺度不变。这是丢失期"只追位置不追尺度"的关键技巧。

> 注意：`update()` 内部依赖 `state_pre_`（上一次 `predict()` 的结果），所以调用顺序必须是 `predict()` → `update()`。SOT 的 `track` 中每帧先 `predict` 再 `update`，符合该约定；跳帧补偿段也成对调用。

## 七、FearTrack 模型后处理

`FearTrack : BaseModel`，双输入（模板 + 搜索）双输出（regression + score）。

```
onModelOpened(): 校验输入层==2, 输出层==2

构造: mean={123.675,116.28,103.53}, std={58.395,57.12,57.375}, rgb_order="rgb"
      (ImageNet 标准化)
      makeGrid(): 16×16 网格, grid_x[y][x]=x*16, grid_y[y][x]=y*16

outputParse(images[batch][2], out):
  for each batch b:
    取 regression_tensor (4×16×16) 与 score_tensor (16×16)
    # 1. score map 上 argmax → (max_i, max_j, max_score)
    #    支持 INT8/UINT8(乘qscale)/FP32
    # 2. 在 max 位置取 4 通道回归值 reg_x1,reg_y1,reg_x2,reg_y2
    #    bbox = [grid_x - exp(reg_x1), grid_y - exp(reg_y1),
    #            grid_x + exp(reg_x2), grid_y + exp(reg_y2)]
    #    (距网格点的距离, 用 exp 保证正值)
    → ModelBoxInfo{bboxes:[{x1,y1,x2,y2,score}], image_width/height=搜索图尺寸}
```

输出框坐标系是 **instance_size_ (256) 空间**，由 SOT 的 `track` Step 3 映射回原图。回归用 `exp(dist)` 表示到网格点的距离，是 anchor-free 的常见做法。

`score_size_=16, total_stride_=16` ⇒ 搜索图 256/16=16，与网格一致。

## 八、C API 与示例用法

### 8.1 C 接口

```c
// 1. (可选) 启用卡尔曼
TDL_SetSingleObjectTrackingUseKalman(handle, true);

// 2. 设置追踪目标 (首帧)
//    size=4: (x1,y1,x2,y2); size=2: (x,y)点; size=1: 检测框索引
//    frame_type: 0/1/2/3 见第三章
//    model_path: frame_type==3 时传 FastSAM 模型路径, 否则 NULL
TDL_SetSingleObjectTracking(handle, img, &obj_meta,
                            set_values, size, frame_id,
                            frame_type, model_path);

// 3. 后续每帧追踪
TDL_SingleObjectTracking(handle, img, &track_meta, frame_id);
// track_meta: status!=LOST 时填充 id + bbox + score
```

C API 内部：首次调用 `TDL_SetSingleObjectTracking` 时惰性创建 `TDL_SOT` 并 `setModel(TDL_MODEL_TRACKING_FEARTRACK)`。`TDL_SingleObjectTracking` 仅调用 `tracker->track`，若 LOST 则不填充结果。

### 8.2 C++ 示例 `sample_img_sot.cpp`

三状态机：`DETECTION` →（按 'i'）`WAITING_INPUT` → `TRACKING` →（丢失>5s）`DETECTION`。

```
main:
  model_factory.loadModelConfig(); setModelDir(model_dir)
  sot_model = getModel(sot_model_id)        // TRACKING_FEARTRACK
  det_model = getModel(det_model_id)        // 任意检测模型
  tracker = TrackerFactory::createTracker(TDL_SOT)
  tracker->setModel(sot_model)

  for each frame:
    按 'i' → WAITING_INPUT: 读 (x,y) 或 (x1,y1,x2,y2)
              det_model->inference → 检测框
              tracker->initialize(image, det_bboxes, point/bbox, 0, frame_type=0)
              → TRACKING
    TRACKING: tracker->track(image, frame_id, info)
              打印 bbox; LOST 计时, >5s 回 DETECTION
```

示例中 `frame_type=0`（不框选，直接用检测框/用户框），是最快路径。

## 十、优化建议（面向后续工作）

下面按"收益/风险"梳理可优化点，供排期参考。

### 10.1 正确性 / 健壮性

1. **`reappear_score_ratio_threshold_` 类型为 `int`（值为 0.3 截断为 0）**
   - 位置：`sot.hpp:97`。声明 `int reappear_score_ratio_threshold_ = 0.3;` 实际恒为 0。
   - 影响：`reappear_score_ratio_abs = max(score_ratio - 0, 0) = score_ratio`，重现判定比预期宽松，可能误重捕获。
   - 建议：改为 `float`。**低风险高收益**。

2. **模板图永不更新**
   - 目标外观剧烈变化（形变、遮挡后重新出现的新视角）时模板过时，长时跟踪易丢。
   - 可选：在 `status_==TRACKED && !is_occluded && score>high_thresh` 且距上次更新超过 N 帧时，用 `current_bbox_` 重新提取模板（带置信度门限与平滑，避免抖动）。需评估 FearTrack 对在线模板更新的兼容性。

3. **几何异常检测阈值固定**
   - `2*prev_width` 位移、`[0.8,1.25]` 面积/宽高比区间为硬编码，对快速运动/小目标不友好。
   - 可按目标尺寸自适应（位移阈值 ∝ 对角线长度），或仅在卡尔曼预测偏离大时触发。

4. **`getStatus` 权重为经验魔法数**
   - `0.1/0.7/0.5/0.5` 与 `2/2/0.1/-0.2` 未标定。建议抽为可配置参数（已有 `occluded_threshold_` 等部分），并补一组离线标定脚本。

5. **`score_lst_` 队列满才计算 ratio**
   - 前 10 帧无 `score_ratio`（保持初值 1.0），遮挡检测在冷启动期失效。可改为队列未满时用已有均值。

6. **`assert_occluded` 与状态机耦合**
   - `score<0.1 || iou<0.2` 直接置 LOST，与 `is_occluded` 的加权判定并列，可能互相覆盖。建议统一为单一置信度门限或明确优先级文档化。

### 10.2 性能

7. **`FastSAMSegmentor` 与 SOT 推理在同一线程**
   - initialize 阶段若 `frame_type==3`，FastSAM 推理串行；track 阶段每帧一次 FearTrack 推理。可考虑模板/搜索预处理与推理流水化（VPSS 与 TPU 并行）。

8. **预处理 `preprocess` 每帧构造 `PreprocessParams` 并 memset**
   - 小优化：缓存 params，仅更新 crop 字段。

9. **`updateScoreLst` 每次 `std::accumulate` 全量求和**
   - 可维护滚动和 O(1)。收益小但量大。

10. **GrabCut 多半径试探**
    - `frame_type==1` 时 {3,5,11,13} 逐个跑 GrabCut，最坏 4 次 OpenCV 迭代，耗时高。仅 initialize 调用一次，但若频繁重初始化（sample 中丢失>5s 回检测）会成为瓶颈。可缓存或限定单半径。

11. **`preprocessor_` 用 `AUTOMATIC` 平台**
    - 模板/搜索预处理若能在 TPU/VPSS 上做（BMCV/VPSS）而非 CPU OpenCV，端侧延迟能显著下降。确认 `PreprocessorFactory` 在目标平台是否真的落到硬件加速。

### 10.3 工程 / 可维护

12. **大量硬编码阈值散落在头文件**
    - 建议统一到 `TrackerConfig` 或 SOT 专属 config 结构，支持运行时注入（json），便于调参与 A/B。

13. **`sot.cpp` 中 GrabCut/Color/FastSAM 三段框选代码在两个 `initialize` 重载里几乎完全重复**
    - (bbox 模式与 point 模式各有一份) 应抽取 `selectBySegmentation(image, seed, frame_type, model_path)` 公共方法，减少维护成本与不一致风险。**中风险中收益（重构）**。

14. **`SOTInfo` 部分字段未真正使用**
    - `bbox/kalman_bbox/size_ratio` 等写入后主要用于 LOGI 调试。可精简或明确为调试专用（编译开关）。

15. **日志大量 `LOGI` 浮点格式输出**
    - Release 默认 `DISABLE_LOG`，但 Debug/Asan 下每帧多条 LOG 影响时序。可分级。

16. **`std::cout` 残留**
    - `sot.cpp` 中 FastSAM 分支有 `std::cout << "area_bbox: ..."`，应改为 LOGI 或删除。

### 10.4 优化优先级建议

| 优先级 | 项 | 理由 |
|--------|-----|------|
| P0 | #1 `int` 阈值 bug | 明确缺陷，一行修复 |
| P0 | #13 框选代码去重 | 降低后续优化改动风险 |
| P1 | #12 参数可配置化 | 为后续调参/标定打基础 |
| P1 | #2 在线模板更新 | 长时跟踪精度核心瓶颈 |
| P1 | #3/#4 自适应阈值与权重标定 | 直接影响跟踪稳定性 |
| P2 | #11 硬件加速预处理 | 端侧延迟 |
| P2 | #5/#6 冷启动与状态判定 | 边界场景 |
| P3 | #7/#10 并行化/GrabCut | 性能锦上添花 |
