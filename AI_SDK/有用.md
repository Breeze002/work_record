
● sample/ 的目录是这样组织的——按语言分顶层,每种语言内再按运行场景分文件夹:
  
  顶层:按语言分

  sample/
  ├── c/         纯 C 接口示例
  ├── cpp/       C++ 接口示例(最全)
  └── python/    Python 接口示例

  三种语言对应 SDK 的三套对外接口,功能类似但覆盖面不同(C++ 最完整)。

  各语言内的子文件夹

  sample/cpp/

  ┌──────────┬──────────────────────────────────────────────────────────────────────────────────────┐
  │  子目录  │                                         作用                                         │
  ├──────────┼──────────────────────────────────────────────────────────────────────────────────────┤
  │ app/     │ 完整业务应用(多模型组合 + 业务逻辑),如人脸抓拍、跌倒检测、ADAS、客流计数、身份识别等 │
  ├──────────┼──────────────────────────────────────────────────────────────────────────────────────┤
  │ camera/  │ 基于 VI(视频输入)硬件的实时示例,如 sample_vi_md(VI 运动检测)                         │
  ├──────────┼──────────────────────────────────────────────────────────────────────────────────────┤
  │ config/  │ app 的 JSON 配置文件,每个 app 对应一份(模型路径、参数、阈值等)                       │
  ├──────────┼──────────────────────────────────────────────────────────────────────────────────────┤
  │ (根目录) │ 大量单功能 sample_*.cpp,逐个演示某个模型/功能                                        │
  └──────────┴──────────────────────────────────────────────────────────────────────────────────────┘

  sample/c/

  ┌─────────┬─────────────────────────────────────────────────────────────────────────────────────────────┐
  │ 子目录  │                                            作用                                             │
  ├─────────┼─────────────────────────────────────────────────────────────────────────────────────────────┤
  │ app/    │ C 版的完整应用(adas、跌倒、人体姿态、人脸宠物抓拍等)                                        │
  ├─────────┼─────────────────────────────────────────────────────────────────────────────────────────────┤
  │ camera/ │ C 版 VI 实时流示例,最丰富——sample_vi_*.c 演示从摄像头取流跑检测/跟踪/分割/编码/RTSP 等      │
  ├─────────┼─────────────────────────────────────────────────────────────────────────────────────────────┤
  │ config/ │ C 版 app 的 JSON 配置(带 _vi 后缀,表示面向 VI 场景)                                         │
  ├─────────┼─────────────────────────────────────────────────────────────────────────────────────────────┤
  │ utils/  │ C 示例的公共工具:可视化(meta_visualize)、线程工具(pthread_utils)、通用 helper(sample_utils) │
  └─────────┴─────────────────────────────────────────────────────────────────────────────────────────────┘

  sample/python/

  ┌───────────┬──────────────────────────────────────────────┐
  │  子目录   │                     作用                     │
  ├───────────┼──────────────────────────────────────────────┤
  │ Qwen/     │ Qwen 大模型示例(sample_qwen.py + token 配置) │
  ├───────────┼──────────────────────────────────────────────┤
  │ Qwen2_VL/ │ Qwen2-VL 多模态示例(pipeline.py + 拉流 demo) │
  ├───────────┼──────────────────────────────────────────────┤
  │ (根目录)  │ 各任务的单文件 .py 示例                      │
  └───────────┴──────────────────────────────────────────────┘

  一句话理解层级关系

  同一类示例通常有三个层次的版本,逐步贴近真实部署:

  1. 单图/单模型版(根目录 sample_img_*.cpp):喂一张图,跑一个模型——验证模型本身
  2. MP4/音频流版(sample_mp4_*.cpp、sample_audio_*.cpp):喂文件流——验证多帧流水线
  3. VI 摄像头版(camera/sample_vi_*.c):从真实硬件取流——验证端侧实时部署
  4. App 版(app/):多模型组合 + 业务逻辑 + 配置——可直接交付的落地方案
