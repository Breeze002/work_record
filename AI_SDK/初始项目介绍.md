# tdl_sdk 项目全面解析文档

> 本文档基于源码梳理，全面介绍 `tdl_sdk`（Turnkey Deep Learning SDK）的整体架构、代码结构、核心模块、工作流程及典型用法，包含工作流程图与代码结构图。

---

## 一、项目概述

### 1.1 项目定位

`tdl_sdk` 是一个基于**算能（Sophgo）芯片产品**的开箱即用深度学习算法 SDK，致力于为用户提供跨平台（端、边）、简单易用、资源节约、性能高效的算法库及应用。

核心目标：
- **一次部署，多平台运行**：基于统一底座框架适配多种芯片。
- **充分利用硬件加速单元**：TPU、VPSS、IVE 等实现高效推理与图像处理。
- **内存高效**：避免冗余内存申请与拷贝。
- **多语言接口**：提供 C / C++ / Python 三种接口，方便集成。

### 1.2 支持的平台

| 平台枚举 | 芯片系列 | 角色 |
|----------|----------|------|
| `CV180X` / `CV181X` | CV180X / CV181X | 端侧（SoC） |
| `CV184X` | CV184X | 端侧 |
| `88a2` | CV186AH | 端侧 |
| `BM1688` / `BM1684X` / `BM1684` | BM168X 系列 | 边侧（PCIE/SoC） |
| `CMODEL_CV181X` / `CMODEL_CV184X` | CModel 仿真 | x86_64 开发测试 |

编译宏映射（见 `CMakeLists.txt`）：
- `__CV180X__` / `__CV181X__` / `__CV184X__` / `__CV186X__`：端侧
- `__BM168X__` + `__BM1688__` / `__BM1684__` / `__BM1684X__`：边侧
- `__CMODEL_CV181X__` / `__CMODEL_CV184X__`：仿真
- `__EDGE_PLATFORM__`：BM168X 系列统一定义

### 2.2 设计原则

- **模块化**：同类功能抽象基类，高内聚低耦合；仅暴露基类，不暴露实现。
- **可扩展**：通过继承基类实现新功能（新模型、新平台预处理等）。
- **可维护**：尽量不引入复杂设计，参考 MediaPipe 的 Packet 思想但更轻量。

```
   │  free_queue_  →[Node0]→[Node1]→[Node2]→...→ final_queue_ │

   └─────────────────────────────────────────────────────────┘
                │  每个节点:
                ▼
   PipelineNode(worker, max_pending_frame)
   ├─ worker_: Packet (工作实例, 如 BaseModel / Tracker)
   ├─ input_queues_: map<channel*, BlockingQueue<PtrFrameInfo>>
   ├─ process_func_: 外部传入的处理函数
   ├─ thread_: 工作线程 (process 静态函数)
   ├─ channels_: 该节点所属通道列表

                │
                ▼

     frame_id_, frame_width, frame_height,
     node_data_: map<string, Packet>   // 各节点输出存这里
   }
   Packet: 类型擦除的数据包 (借鉴 MediaPipe)

   NodeFactory::createModelNode(model)  → 单例: 同一 model 共享同一节点
```

## 四、核心工作流程

### 4.1 整体编译工作流

```
   build_tdl_sdk.sh <platform> [sample|all] [debug|static|...]
            │
            ▼
   1. source scripts/credential.sh  (FTP 第三方库凭证)
            │
            ▼
   2. 设置环境变量: CVI_PLATFORM, TOOLCHAIN, OPENOPENCV, ...
            │
            ▼
   3. 调用 cmake -DCMAKE_TOOLCHAIN_FILE=... -DCVI_PLATFORM=... -DBUILD_SHARED=ON
            │
            ▼
   ┌───────────────────────────────────────────┐
   │ CMakeLists.txt 解析流程:                  │
   │  a. 平台宏定义 (__CV181X__ 等)            │
   │  b. download_thirdparty.sh (按需)         │
   │  c. parse_build_config.py 解析 build_config.yaml
   │     → cmake/build_config.cmake (模块开关) │
   │  d. include cmake/opencv|mlir|middleware|  │
   │        thirdparty|ive|cvi_rtsp.cmake      │
   │  e. add_subdirectory(src/sample/tests/...)│
   └───────────────────────────────────────────┘
            │
            ▼
   4. make -j$(N)  → 产物到 install/(lib,include,configs,scripts)
            │
            ▼

```

`build_config.yaml` 模块开关（`ENABLE_SLIM=1` 时生效）：
```

ENABLE_KEYPOINTS_DETECTION / ENABLE_LICENSE_PLATE_RECOGNITION
ENABLE_OBJECT_DETECTION / ENABLE_OBJECT_TRACKING / ENABLE_SEGMENTATION
ENABLE_APP_PIPELINE ...
```

### 4.2 单模型推理工作流（底座框架核心）

```
用户调用 BaseModel::inference(images, out_datas)
        │
        ▼
┌──────────────────────────────────────────────────────────┐
│ 1. 取输入层名 input_layer_name = net_->getInputNames()[0] │
│    取该层 PreprocessParams                                  │
└──────────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────────┐
│ 2. 分批处理 (batch)                                        │
│    while (process_idx < batch_size):                       │
│      fit_batch = getFitBatchSize(剩余)                     │
│      setInputBatchSize(input_layer_name, fit_batch)        │
└──────────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────────┐
│ 3. 对 batch 中每张图预处理:                                │
│    preprocessor_->preprocessToTensor(                      │
│        image, preprocess_params, i, input_tensor)          │
│    └─> VpssPreprocessor / OpencvPreprocessor / BmcvPreproc │
│    同时记录 rescale_params (用于后处理坐标还原)            │
└──────────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────────┐
│ 4. net_->updateInputTensors()  (同步 cache + 准备输入)     │
│    net_->forward()             (TPU 推理)                  │
│    net_->updateOutputTensors() (取回输出)                  │
│    └─> CviNet (端侧 MLIR) / BmNet (边侧 BMRuntime)         │
└──────────────────────────────────────────────────────────┘
        │
        ▼
┌──────────────────────────────────────────────────────────┐
│ 5. outputParse(batch_images, batch_results)  [虚函数]     │
│    由子类实现，从输出 Tensor 解析为 ModelOutputInfo        │
│    例: RgbImageClassification → ModelClassificationInfo    │
│        YoloV8 → ModelBoxInfo[]  (检测框)                   │
│        Scrfd → ModelFaceInfo[] (人脸)                      │
│        YoloV8Seg → ModelInstanceSegInfo[]                  │
└──────────────────────────────────────────────────────────┘
        │
        ▼
   out_datas 返回给用户
```

预处理参数（与训练对齐）：
- `Y = X * scale - mean`
- `scale_tdl = 1/std`，`mean_tdl = -mean/std`
- 支持 `keep_aspect_ratio`（等比缩放居中）、crop、PLANAR/PACKED 转换。

### 4.3 模型实例化工作流（ModelFactory）

```
   TDLModelFactory::getInstance()
        │
        ├─ loadModelConfig("model_factory.json")   // 解析 model_list
        │      │
        │      ▼ model_config_map_[name] = json
        │
        ▼ getModel(ModelType::YOLOV8N_DET_HAND, path?, config?, dev?)
   ┌──────────────────────────────────────────────┐
   │ getModelImpl:                               │
   │  1. 确定平台 platform_ + 模型后缀 (.cvimodel/.bmodel) │
   │  2. getModelInstance(type)                  │
   │     └─ isXxxModel(type) → createXxxModel()  │
   │        返回具体子类实例 (如 YoloV8)          │
   │  3. 设置 net_param_.model_path / pre_params │
   │     (优先用 json config, 其次子类默认值)    │
   │  4. NetFactory::createNet(platform) → net_  │
   │  5. PreprocessorFactory::createPreprocessor │
   │  6. model->modelOpen(device_id)             │
   │     └─ net_->loadModel() + onModelOpened()  │
   └──────────────────────────────────────────────┘
        │
        ▼ 返回 shared_ptr<BaseModel> (已打开, 可直接 inference)
```

模型路径解析：默认从 `parent_dir_of(tdl.so)/configs/model/model_factory.json` 加载配置；`model_dir_` 指向 `tdl_models` 仓库（各平台子目录）。`github_url` 指向 <https://github.com/sophgo/tdl_models>。

```
│ TDL_Detection/Face/...    │  │ out = m.inference(img)      │
│ TDL_APP_Init/SetFrame/... │  │                             │
│ TDL_Tracking/SingleTrack  │  │ 模块: image / nn / matcher  │
│ TDL_DestroyHandle         │  │       / llm                 │
└───────────────────────────┘  └─────────────────────────────┘
            │                              │
            └──────────┬───────────────────┘
                       ▼
            共用同一套底座框架 + 组件实现 (libtdl.so)
```

C API 设计要点：
- `TDLHandle` 持有 device_id 与已加载模型表（按 `TDLModel` 索引）。
- `TDLImage` 包装 `BaseImage`（可由 `VIDEO_FRAME_INFO_S` wrap，支持 `own_memory`/`is_preprocessed`）。
- 专有模型可传 `NULL` config（用类内默认）；通用模型（特征提取、声音指令）需传 json config。
- APP API（`TDL_APP_Init/SetFrame/Capture/...`）封装多模型协同任务。

### 4.5 Pipeline 多模型协同工作流（以人脸抓拍为例）

```
   TDL_APP_Init(handle, "face_capture", json_config, &names, &size)
        │
        ▼
   AppFactory::createAppTask("face_capture", json)
        │
        ▼
   FaceCaptureApp::init()  (子类实现)
        │
        ▼
   构建 PipelineChannel（每路视频一个 channel）
        │
        ▼
   串联节点:
   ┌────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
   │ 解码   │ → │ 人脸检测 │ → │ 人脸跟踪 │ → │ 质量评估 │ → │ 特征提取 │
   │ Node0  │   │ (单例)   │   │          │   │ /抓拍    │   │ /匹配    │
   └────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
        │            │              │              │              │
        │      各节点独立工作线程, 从自身 input_queues_ 取帧
        │      处理完通过 channel->toNextNode() 流转到下一节点
        │      数据挂在 frame->node_data_[name] (Packet)
        ▼
   TDL_APP_SetFrame(handle, channel_name, image, frame_id, buffer_size)
        │  (外部送帧到 channel 的 free_queue_ → 第一个节点)
        ▼
   ... 异步处理 ...
        │
        ▼
   TDL_APP_Capture(handle, channel_name, &capture_info)
        │  (从 channel->final_queue_ 取完成帧, 解析抓拍结果)
        ▼
   TDLCaptureInfo { 最佳人脸图, track_id, 特征, 属性, ... }
```

节点特性：
- **单例节点**：模型节点（`NodeFactory::createModelNode` 同一 model 返回同一 node），多通道共享。
- **非单例节点**：解码、跟踪等每通道独立。
- 节点间通过 `PtrFrameInfo`（含 `map<string,Packet>`）传递，互不干扰。

### 4.6 单目标跟踪（SOT）工作流（用户当前打开的 sot.cpp）

```
   TrackerFactory::createTracker(TDL_SOT) → SOT 实例
        │
        ▼
   SOT::initialize(image, detect_boxes, bbox/point/index, frame_id, frame_type, model_path)
   ┌────────────────────────────────────────────────────────────┐
   │ 1. 若有检测框:                                            │
   │    - bbox 模式: 找中心点落入 bbox 的最近检测框            │
   │    - point 模式: 找包含该点的检测框                       │
   │    - index 模式: 直接取 detect_boxes[index]               │
   │ 2. 若无匹配检测框 / 无检测框:                             │
   │    按 frame_type 用目标框选算法获取更紧致框:              │
   │      1=GrabCut  2=Color  3=FastSAM  0=直接用原框          │
   │ 3. initBBox(image, init_bbox):                            │
   │    - clampBBox 边界约束                                   │
   │    - 当前框 = 模板框 = last_reliable_template_bbox        │
   │    - (可选) KalmanBoxTracker 初始化                       │
   │    - preprocess 提取模板图 (template_image_, context 区域)│
   └────────────────────────────────────────────────────────────┘
        │
        ▼
   SOT::track(image, frame_id, tracker_info)  —— 每帧调用
   ┌────────────────────────────────────────────────────────────┐
   │ 1. preprocess 提取搜索图 (search_image, 基于 current_bbox) │
   │ 2. sot_model_->inference({{template, search}}, out)        │
   │    └─ 双输入孪生网络, 输出 ModelBoxInfo                    │
   │ 3. 坐标映射回原图 (按 context 缩放) + clampBBox            │
   │ 4. 几何异常检测:                                           │
   │    中心位移 > 2*宽  OR  面积比 ∉[0.8,1.25] OR 宽高比异常   │
   │    → 判定 LOST                                             │
   │ 5. (可选) 卡尔曼预测 + IoU 计算                            │
   │ 6. updateScoreLst: 维护近 10 帧得分, 算 score_ratio        │
   │ 7. getStatus(): 加权得分判断 遮挡/重现 状态机              │
   │    confidence_of_occluded  = 0.1*score_abs + 0.7*score_ratio_abs
   │                            + 0.5*iou_abs + 0.5*size_ratio_abs
   │    confidence_of_reappear  = 2*reappear_score_abs + ...    │
   │ 8. 状态更新:                                              │
   │    - TRACKED & 未遮挡: 更新卡尔曼(观测), 更新可靠模板框    │
   │    - 遮挡: 置 LOST, 卡尔曼仅预测, lost_frames++, 逐帧扩张搜索框
   │    - LOST & 重现: 恢复 TRACKED                            │
   │ 9. 输出 tracker_info{box_info, status_}                   │
   └────────────────────────────────────────────────────────────┘
```

### 4.7 多目标跟踪（MOT）工作流

```
   TrackerFactory::createTracker(TDL_MOT_SORT) → MOT 实例
        │
        ▼
   MOT::track(boxes, frame_id, trackers)
   ┌────────────────────────────────────────────────────────────┐
   │ 1. 所有 KalmanTracker 预测新位置                          │
   │ 2. 构造代价矩阵 (检测框 vs 跟踪框, 基于 IoU/距离)         │
   │    └─ CostMatrixHelper                                    │
   │ 3. Munkres (匈牙利算法) 最优匹配                          │
   │ 4. 匹配成功: 更新对应 KalmanTracker                       │
   │    未匹配检测: 新建 Tracker                               │
   │    未匹配跟踪: 标记 lost, 超阈值则删除                    │
   │ 5. 输出 trackers (含 track_id + box)                      │
   └────────────────────────────────────────────────────────────┘
```

## 五、关键数据类型与配置

### 5.1 图像类型系统

- **ImageFormat**：GRAY / RGB/BGR_PLANAR/PACKED / YUV420SP(NV12/NV21) / YUV420P(I420/YV12) / YUV422* / UNKOWN
- **TDLDataType**：INT8/UINT8/INT16/UINT16/INT32/UINT32/BF16/FP16/FP32
- **ImageType**：VPSS_FRAME / OPENCV_FRAME / FFMPEG_FRAME / BMCV_FRAME / TENSOR_FRAME / RAW_FRAME
  - `TENSOR_FRAME`：可复用图像内存作为输入 tensor（`setInputTensorFromImage`），避免重复分配。

### 5.2 模型输出类型（ModelOutputInfo）

| 子类 | 用途 |
|------|------|
| `ModelBoxInfo` | 检测框（x1,y1,x2,y2,score,class_id） |
| `ModelFaceInfo` | 人脸框 + 关键点 |
| `ModelClassificationInfo` | 分类（类别、置信度） |
| `ModelInstanceSegInfo` | 实例分割（框 + mask） |
| `ModelSegInfo` | 语义分割 |
| `ModelLandmarkInfo` / `ModelKeypoint` | 关键点 |
| `ModelFeatureInfo` | 特征向量 |
| `ModelLaneInfo` | 车道线 |
| `ModelTextInfo` | OCR / 语音识别文本 |
| `ModelDepthInfo` | 深度估计 |
| `ModelBox3DInfo` | 3D 框（ADAS） |

### 5.3 模型配置（model_factory.json）

每个模型条目示例：
```json
"YOLOV8N_DET_HAND": {
  "types": ["hand"],
  "file_name": "yolov8n_det_hand_384_640_INT8",
  "rgb_order": "rgb"        // 可选, 默认按模型
}
```
- `types`：类别名列表，与 `tdl_model_list.h` 中注释对应，用于 `type_mapping_`。
- `file_name`：模型文件名（不带平台后缀，工厂按平台自动补 `.cvimodel`/`.bmodel`）。
- 通用模型（CLS_IMG / 特征提取等）还可在此指定 `mean`/`std`/`rgb_order`/`img_format`。

### 5.4 模块编译开关（build_config.yaml）

通过 `ENABLE_SLIM=1` 启用按需编译，未启用的模块不编入 `libtdl.so`，减小体积。注意：`APP_PIPELINE` 需 `OBJECT_DETECTION`、`FACE_DETECTION`、`FACE_LANDMARK` 同时开启。

## 六、典型使用示例

### 6.1 C++ 单模型推理（最简）

```cpp
#include "nn/tdl_model_factory.hpp"
#include "framework/image/image_factory.hpp"

int main() {
  auto& factory = TDLModelFactory::getInstance();
  factory.loadModelConfig();  // 可选, 默认从 install 路径加载
  auto model = factory.getModel(ModelType::YOLOV8N_DET_HAND, "model_path");
  auto img = ImageFactory::readImage("test.jpg");
  std::vector<std::shared_ptr<ModelOutputInfo>> out;
  model->inference({img}, out);
  // out[0] -> ModelBoxInfo, 取 bboxes
}
```

### 6.2 C 接口

```c
TDLHandle h = TDL_CreateHandle(0);
TDL_LoadModelConfig(h, NULL);
TDL_OpenModel(h, TDL_MODEL_..., "model_path", NULL, 0);
TDLImage img = TDL_ReadImage("test.jpg");
TDLObject obj;
TDL_Detection(h, TDL_MODEL_..., img, &obj);
TDL_DestroyImage(img);
TDL_DestroyHandle(h);
```

### 6.3 Python 接口

```python
from tdl import nn, image
detector = nn.get_model(nn.ModelType.SCRFD_DET_FACE, "model_path")
img = image.read("test.jpg")
bboxes = detector.inference(img)
```

### 6.4 App 多模型任务（C 接口）

```c
char** names; uint8_t size;
TDL_APP_Init(h, "face_capture", "face_cap_config.json", &names, &size, false);
// 循环送帧
TDL_APP_SetFrame(h, names[0], img, frame_id, buffer_size);
TDLCaptureInfo cap;
TDL_APP_Capture(h, names[0], &cap);
```

### 7.1 构建命令

```bash
# 端侧 SoC (例 CV181X)
./build_tdl_sdk.sh CV181X

# 调试 / 静态
./build_tdl_sdk.sh CV181X debug
```

构建类型：`Release`(默认,关log) / `Debug` / `Asan` / `Tsan` / `UBsan` / `SDKRelease`。

### 7.2 部署产物

`install/` 目录：`lib/libtdl.so` + `include/` + `configs/` + `scripts/envsetup.sh`。运行前 `source scripts/envsetup.sh` 配置 `LD_LIBRARY_PATH`。

### 7.3 模型获取

模型仓库：<https://github.com/sophgo/tdl_models>，按平台子目录存放。`TDL_SetModelDir` 或 `model_factory.json` 指定路径。

## 十、上手建议路径

1. **理解分层**：先看本文档第二章分层架构，建立“底座框架→组件→App”的心智模型。
2. **跑通 sample**：从 `sample/cpp/sample_img_object_detection.cpp` / `sample_img_face_detection.cpp` 起步，编译运行（`docs/getting_started/build.md` + `run.md`）。
3. **理解推理流**：精读 `include/framework/model/base_model.hpp` 与 `src/framework/model/base_model.cpp` 的 `inference`，对照第四章 4.2 节流程。
4. **理解模型工厂**：读 `tdl_model_factory.cpp` 与 `model_factory.json`，掌握“按 ModelType + 平台”实例化机制（4.3 节）。
5. **进阶 Pipeline/App**：读 `pipeline_node.hpp` / `pipeline_channel.hpp` 与任一 App（如 `face_capture_app`），对照 4.5 节。
6. **专项深入**：按需阅读跟踪（SOT/MOT，4.6/4.7）、语音（VAD/ASR/Zipformer）、LLM（Qwen）、媒体分析服务等。
7. **接入新模型**：参考 `docs/tutorials/integrate_new_model.md` 与 `docs/developer_guide/yolo_development_guide.md`。
